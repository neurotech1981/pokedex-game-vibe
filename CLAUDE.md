# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev       # Start Vite dev server
npm run build     # tsc && vite build
npm run lint      # ESLint (ts/tsx, --max-warnings 0)
npm test          # Vitest (tests live in src/utils/__tests__/)
npm run deploy    # Build + publish dist/ to gh-pages branch
npm run fetch:assets  # Re-download bundled gen 1–3 assets (sprites/artwork/cries) via git sparse clone
```

Run a single test file: `npx vitest run src/utils/__tests__/battleEngine.test.ts`

Deployed to GitHub Pages; `vite.config.ts` sets `base: '/pokedex-game-vibe/'`. Asset URLs must use `import.meta.env.BASE_URL` (see `src/utils/soundEffects.ts`).

## What this app is

React 19 + TypeScript + Vite Pokédex/battle simulator using MUI v7, Emotion, framer-motion, and TanStack React Query v5. Installable PWA via `vite-plugin-pwa` in `vite.config.ts` (autoUpdate SW; precache = app shell only — audio/backgrounds/particles/trainers are runtime-cached CacheFirst, PokeAPI is NetworkFirst; icons `public/pwa-*.png` are generated flat-SVG rasters, NOT the 30×30 `assets/items/poke-ball.png`). `registerSW` is called in `main.tsx`; `vite-env.d.ts` references `vite-plugin-pwa/client`.

## Architecture

Single component tree, no router, no global state library:

```
main.tsx  → QueryClientProvider + MUI ThemeProvider (src/theme.ts)
App.tsx   → <Pokemon />
Pokemon.tsx (~1500 lines) → 6 tabs:
  Pokédex            (list/detail/evolutions, favorites, compare)
  TeamBuilder.tsx    (teams persisted to localStorage key "pokemonTeams")
  TypeEffectiveness.tsx
  BattleSimulator.tsx (~3000 lines; receives teams, getTypeColor, TYPE_EFFECTIVENESS as props)
  TrainerCard.tsx    (records/dex/badges/achievements/save-data/replays/Box)
  PokeMart.tsx       (spend PokéCoins: items/balls/held items/vitamins + daily reward)
```

### Battle system (the core of the app)

- **Rules engine**: `src/utils/battleEngine.ts` — pure, immutable, rng-injectable. `resolveAction(state, action, chart, rng)` returns `{ state, events: BattleEvent[] }`; the UI never mutates battle state directly. Mechanics: stat stages (±6, `stageMultiplier`), speed-based round order (`beginBattle`, faster active mon leads each round; paralysis halves speed; Quick Claw can steal the lead), physical/special split (`getDamageClass`, Gen-3 type rules; special moves read special-attack/special-defense; burn weakens physical only), weather/terrain, status effects incl. confusion self-hits, energy system, abilities (Overgrow/Blaze/Torrent low-HP boost; Intimidate on-enter; Levitate immunity; Static retaliation; Sturdy survive-at-1HP), and held items (`src/data/items.ts` `HELD_ITEMS`: Leftovers round heal, type boosters, Focus Sash, Quick Claw). `BattleMon.moves` carries the moveset (defaults to type-derived). Tested in `src/utils/__tests__/` (engine, AI, movesets, progression, gauntlet, recruitment; shared fixtures in `helpers.ts`).
- **Movesets**: `src/utils/movesets.ts` — real per-Pokémon level-up learnsets from PokeAPI (`getMovesetForPokemon`, cached per species + localStorage mirror `pokedexGame.movesets.v2`); any fetch failure falls back to `getMovesForTypes`. `getFullLearnset` (method-tagged, ≤20 level-up + ≤20 machine/tutor, mirror `pokedexGame.learnsets.v2`) feeds the TeamBuilder Move Manager (`MoveManagerDialog`, saves `MonProgress.customMoves`; TM/tutor rows are coin-gated — see EVs & TMs). `mapApiMove` also maps priority / multi-hit / flinch / stat-change debuffs. Battle start is async (`starting` flag guards double-clicks).
- **Combat depth**: priority uses a **momentum model** (a priority move makes you lead the NEXT round — a true interrupt is impossible in the round-first turn model; precedence momentum > Quick Claw > speed in `determineRoundFirst`). Multi-hit loops per-hit damage (survive-once effects check each hit), flinch blocks a defender that hasn't acted this round (`BattleMon.flinched`, consumed on act, cleared on switch/round-end), debuffs lower opponent stages. The `'move'` event carries `damageClass` (drives scene FX). Expert AI predicts lethal threats and retreats to resistant bench mons.
- **Safari & catching**: `src/utils/safari.ts` (biomes → type pools + backdrops, `rollWildEncounter` rarity tiers) and engine support in battleEngine.ts (`EngineState.wild/balls/caught`, `throwBall` action, pure `catchChance`). Caught mons go to the Box; ball economy on `profile.balls` (`BALLS` in data/items.ts, drops in data/rewards.ts).
- **Battle Tower**: `src/utils/tower.ts` — level-50-normalized 3v3 streak ladder (no held items, fixed item loadout — never persists engine items back to the profile), every 7th battle a boss with an elite ace; streaks on `records.towerStreak/towerBestStreak`.
- **Achievements**: `src/utils/achievements.ts` pure catalog + `evaluateAchievements`/`applyAchievements`, run inside useBattleResults after the profile update; earned ids on `profile.achievements`. Trainer Card tab (`TrainerCard.tsx`) renders records/dex completion/badges/achievements/save-data card/replays/Box.
- **Natures & IVs**: `src/data/natures.ts` (25 classic natures ±10%, `Ivs` 0–31/stat, `natureMultiplier`/`rollNature`/`rollIvs`/`natureLabel`). Persisted on `MonProgress.nature/ivs` — rolled ONCE in `registerMonProgress` (rng-injectable last param) and backfilled once for old saves in `mergeProfile` (`usePlayerProfile.ts`, exported + tested; never rerolls filled entries). Applied in `createBattleMon`: `effectiveBase = round((base + floor(iv/2) + floor(ev/4)) * natureMult * statMod)` — HP never nature-modified; no opts → identity (guard covers nature/ivs/evs). Player-side battles thread `progress.nature/ivs/evs` (all six start handlers + hotseat P2 + tower); AI opponents stay neutral. TeamBuilder shows a nature chip + IV/EV tooltip per slot.
- **EVs & TMs (M7)**: `Evs` sparse map + `EV_STAT_CAP` 252 / `EV_TOTAL_CAP` 510 in natures.ts. Sources: **vitamins** (6 in `shop.ts`, +10 EVs via pure `applyVitamin` — partial grants at caps, null when full; used from a TeamBuilder per-slot select, inventory `profile.vitamins`) and a **battle trickle** (`applyBattleEvs`: winners gain kos*4 + survived*2 into their `highestBaseStat`, deterministic, same BattleXpEntry list as XP). Replay-safe for free (effective-pokemon snapshots). **TM/Move Tutor**: learnsets are method-tagged (`LearnsetEntry.method`, mirror bumped to `pokedexGame.learnsets.v2`, ≤20 level-up + ≤20 machine/tutor via `selectLearnsetCandidates`); MoveManagerDialog gates machine (500🪙)/tutor (800🪙) rows behind `MonProgress.unlockedMoves` purchases. Evolution transfers `evs` + `unlockedMoves` (customMoves still deliberately dropped).
- **Pokédex completion**: `profile.dex { seen: number[]; caught: number[] }`. Seen registers in `launchBattle` (both sides, once per battle, skipped for replays); caught registers inside `registerMonProgress` (covers catch/recruit/starter/box→team; caught implies seen). Pure helpers in progression.ts (`registerDexSeen/registerDexCaught` — same-ref when no change, `dexCompletion`, `KANTO_DEX_SIZE`); pre-dex saves backfill caught from mons∪box. UI: header chip + per-card caught/seen icons in the Pokédex tab, progress bars on TrainerCard (Johto/Hoenn rows appear once a region id registers — `HOENN_DEX_MAX` 386), dex achievements (Kanto thresholds + Hoenn badges/champion).
- **Battle replays**: `src/utils/rng.ts` (`mulberry32` seeded PRNG) + `src/utils/replay.ts` (localStorage `pokedexGame.replays.v1`, newest-first cap 20, `format` stamp). The engine rng is seeded per battle (`battleRngRef` in BattleSimulator); EVERY engine call (beginBattle/resolveAction/resolveForcedSwitch — player, AI and forced-switch paths) must use it or replays drift (replay.test.ts proves record→rebuild→re-run gives identical events). Both sides' actions are recorded as `ReplayStep`s — AI is NEVER re-run on playback (`selectAIAction` keeps Math.random). `monConfigFrom` snapshots the EFFECTIVE pokemon object (post nature/IV/statMod rewrite) so playback is exact and offline — rebuild passes NO nature/ivs/statMod to createBattleMon (they're already folded in; passing them would double-apply). Playback: `pendingReplay` prop from Pokemon.tsx (queued by TrainerCard Watch → Battle tab), step-driver effect paces recorded steps; `replaying` guards ALL consequences (useBattleResults, dex-seen, saving, input, AI effects). Debug weather/terrain taints the recording (not saved).
- **Hooks split**: battle presentation lives in `src/hooks/useBattleEvents.ts` (log/floating text/SceneFx/processEvents) and consequences in `src/hooks/useBattleResults.ts` (XP/drops/league/tower/catch/achievements/recruit/evolution — applied exactly once per battle); shared UI constants in `components/battle/battleUi.ts`. BattleSimulator keeps mode glue + start handlers.
- **Cries**: `playCry(pokemonId)` in soundEffects.ts — bundled `assets/cries/{id}.ogg` for gen 1–2 (`localCry`), PokeAPI cries GitHub mirror above 251 (send-out, faint, Pokédex detail button); best-effort, silent on failure.
- **League Challenge**: `src/data/league.ts` — 8 Kanto gyms → Elite Four → Champion Blue, then the **Johto post-game** (8 more gyms → Champion Red at floor 100; beating Red sets `league.champion2`), then the **Hoenn expansion** (8 gyms → Sidney/Phoebe/Glacia/Drake → Champion Steven; ALL Hoenn floors are pinned at the level-100 ceiling — Red already sits on it, so difficulty comes from expert AI/held items/team depth, and league.test asserts per-region floor monotonicity, flat for Hoenn; beating Steven sets `league.champion3`). Gating is pure array order — `isStageUnlocked` requires all prior stages. `GYM_STAGES` is Kanto-only (excludes Johto AND Hoenn sets); `JOHTO_*`/`HOENN_*` id-sets/stage lists for the rest. Strict linear unlocks, level floors 52→100 scaling with player avg, rosters via cached `fetchLeagueTeam` (PokeAPI), XP ×2, badges persisted in `profile.league.defeated`; Round 2 rematches (+15 levels, ×2.5 XP, `league.defeatedRematches`) unlock at champion and stay Kanto-only. Trainer portraits bundled in `public/assets/trainers/` (Showdown). UI: `LeagueCard` in BattleSetup, portrait chip in battle, badge labels in PostBattlePanel.
- **Kanto Journey** (adventure mode): `src/data/journey.ts` — 20-node linear path Pallet Town → Indigo Plateau (`JOURNEY_NODES`: trainers, safari `biomeId`s, `gymIds` linking into the league ladder — journey gym order MUST match ladder order, enforced by journey.test.ts). Progress in `profile.journey` (position/clearedTrainers/starterId); gym completion reads `league.defeated` (never duplicated). UI: `battle/JourneyMap.tsx` fullscreen dialog (starter pick incl. hidden Pikachu at level 5 → creates the `journey-team` via `onCreateTeam` prop from Pokemon.tsx; vertical metro map). The Journey card renders via BattleSetup's `journeyCard` prop so it shows even with ZERO teams (it IS the new-player onboarding). Trainer wins append via `journeyTrainerId` in useBattleResults; travel awards Poké Balls.
- **Mobile (phone breakpoint `down('sm')`)**: battle scene uses portrait-aware sprite positions + fov (aspect < 1 → x·0.4/0.55, fov 56, in `SceneContents`; CatchBallEffect takes positions as props); HUD renders `MonStatusPanel dense` pills; move grid is always 2×2; battle log lives in a bottom-sheet Drawer (`logOpen`); everything fits 390×844 with zero dialog scroll. Pokédex detail modal stacks on xs.
- **Orchestration**: `BattleSimulator.tsx` holds React state, bridges engine events → log/sounds/floating text/3D `SceneFx` in `processEvents`, and owns the battle modes: quick vs AI, gauntlet, and **hotseat PvP** (`opponentKind === 'human'` — both teams take input, AI effects disabled, no XP/rewards, both sides get stock inventories). Battle pacing scales with a persisted speed setting (localStorage `battleSpeed`, 1/1.5/2×).
- **AI opponent**: `src/utils/battleAI.ts` — heuristic move scoring over `mon.moves` with difficulties (beginner/intermediate/expert) and personalities (aggressive/defensive/balanced), item use and expert switching.
- **Economy (M7)**: `src/data/shop.ts` holds EVERY balance number (prices, vitamin/TM costs, daily-reward constants, pure `coinsForBattle` — mode-priced, streak bonus capped 10, gauntlet/tower boss ×2, league rematch ×1.5). `profile.coins` (+`lastDailyClaim`/`dailyStreak`/`vitamins`) auto-backfills via the mergeProfile defaults spread — scalars are free, no SAVE_KEYS change (all inside profile.v1). Payout happens ONCE per battle in useBattleResults beside XP (hotseat/replay guard applies); achievements pay coin tiers via `applyAchievements` (once ever, no retroactive backpay). Spending: pure `spendCoins` (null when short) + `claimDailyReward(profile, todayIso)` — the current day is INJECTED like rng, keep it that way. UI: Poké Mart tab (Pokemon.tsx tab 5), coin chips on PostBattlePanel + TrainerCard.
- **Share codes (M7)**: `src/utils/shareCodes.ts` — lz-string `compressToEncodedURIComponent` around a `{kind, version, data}` envelope (`pokedex-game-team`/`pokedex-game-replay`); decode throws readable errors, points wrong-kind pastes to the right surface, gates replays on `canPlayReplay`. Teams ~0.4KB, replays 2–15KB → clipboard-only, never URLs. UI: ShareCodeDialog/ImportCodeDialog shared components; share buttons in TeamBuilder team headers + TrainerCard replay rows; `copyToClipboard` falls back to manual copy.
- **Meta-game** (all pure + tested): `src/utils/progression.ts` (XP curve from level 50, `PlayerProfile` with mons/items/heldItems/records/box/dex persisted via `src/hooks/usePlayerProfile.ts` to localStorage `pokedexGame.profile.v1`; schema backfill lives in the exported pure `mergeProfile`; the hook is instantiated ONCE in `Pokemon.tsx` and passed as props to TeamBuilder + BattleSimulator — never instantiate it twice), `src/data/rewards.ts` (post-win item drops + rare held-item drops), `src/utils/gauntlet.ts` (endless escalating stages, HP carry-over, boss every 3rd stage with elite shiny mons), `src/utils/recruitment.ts` (post-win recruit offers, elite/legendary rolls), `src/utils/evolution.ts` (PokeAPI chain-walk, level-up evolutions; evolving transfers nature/ivs/heldItem). TeamBuilder hosts the **Box manager** (Available | Box tabs; add-to-team/release/drag) plus per-slot level/shiny/elite/nature badges and held-item equipping.
- **Save safety**: `src/utils/saveData.ts` — `SAVE_KEYS` (every localStorage key the game writes; ADD NEW KEYS HERE), `buildFullSave`/`parseFullSave`/`applyFullSave` (injectable storage, tested). TrainerCard's Save Data card does export (Blob download) and import (file input → typed validation → confirm → apply → `window.location.reload()` so the single profile hook rehydrates).
- **Battle UI components** in `src/components/battle/`: `BattleScene3D` (r3f arena), `BattleSetup` (game-menu layout: header banner with inline record chips → hero "NEXT MATCH" builder on a `backdropSx('city')` Paper with glass team panels, lead-mon 88px preview and the dominant Start button → "GAME MODES" **6-col CSS grid** where `wide` tiles span the row and small tiles sit 3-across on md; injected `journeyCard`/`children` are grid cells directly — no wrapper items), `ModeTile` (backdrop-imaged game-mode tile: accent title, stat chip, controls slot, CTA slot — purely presentational, handlers stay in BattleSimulator), `backdropSx(id)` in `battleUi.ts` (bundled backdrop + dark gradient overlay; `isolation: isolate` keeps dialogs above the ::before), `PostBattlePanel` (results, XP bars, drops, recruit/evolution offers), `MoveSelection`, `BattleLog`, `TerrainParticles`, `RecruitOfferCard`, `EvolutionPrompt`. LeagueCard keeps its rich internals but is styled as the full-width `elite4drake` featured tile.
- Teams changes flow through `Pokemon.tsx` handlers (`onAddPokemonToTeam`, `onEvolvePokemon`) — it owns `teams` and the `pokemonTeams` localStorage persistence.

### Data sources

- Pokémon data comes live from PokeAPI (`https://pokeapi.co/api/v2/`) via `fetch` + React Query inside `Pokemon.tsx` (infinite query for the list; separate queries/fetches for species, evolution chains, forms). No local Pokémon dataset.
- **Bundled gen 1–3 assets** (`scripts/fetch-assets.sh`, run via `npm run fetch:assets` — git sparse clone of the pinned SHA, NOT raw.githubusercontent, which rate-limits per-IP): `public/assets/sprites/{id}.png` + `back/` (static), `sprites/anim/[back/][shiny/]{id}.gif` (showdown animated), `artwork/{id}.png` (official artwork, regular only), `cries/{id}.ogg`. Helpers in `src/utils/spriteSources.ts`: `localStaticSprite`/`localAnimSprite`/`localArtwork`/`localCry` (null for id > 386; `getBattleSprites` also serves bundled anim/static for local ids so direct consumers stay same-origin), `dexSprite(id, apiImage)` for the Pokédex list/detail `image` field. **Never point UI at raw.githubusercontent when a bundled asset exists** — a browse of the list used to trip GitHub's per-IP 429 and break every image on the page.
- Battle sprite ladder (`spriteFallbacks`): gen 1–2 is fully same-origin (bundled anim GIF → bundled static → `front_default`; shiny anim is bundled, shiny static falls back to the regular bundled sprite). Gen 3+ walks the CDN: showdown GIF → gen-5 animated → CDN static → `front_default`, URLs pinned to `SPRITES_SHA` (never `master`; bump deliberately — and re-run fetch:assets when you do). Artwork: bundled for regular gen 1–2, CDN for shiny/gen 3+. `src/hooks/useAnimatedTexture.ts` decodes GIFs with gifuct-js into three.js CanvasTextures, **retries the whole ladder with backoff (1s/3s/8s)** when every rung fails, and **on HTTP 429 skips the remaining CDN rungs**. `<img>` fallbacks (BattleSetup preview, TeamRoster) MUST be one-shot with a `dataset` guard: React's synthetic `onError` refires even after `img.onerror = null`, and re-setting `src` to the same failing URL restarts the load → infinite request loop.
- Moves are single-sourced in `src/data/moves.ts`, keyed by type; a Pokémon's moveset is derived from its types (fallback: Tackle). Items live in `src/data/items.ts`, `TYPE_EFFECTIVENESS` in `src/data/typeChart.ts` (a copy still exists in `Pokemon.tsx`/`TeamBuilder.tsx` — passed as props).

### Presentation subsystems

- Move impact FX: `src/utils/moveAnimationHelper.ts` maps all 18 move types → grid spritesheets under `public/assets/particles/Spritesheets/` (some reused with tints); rendered by `ImpactEffect` inside `battle/BattleScene3D.tsx` (texture-offset animation, additive blending; critical hits add a gold burst overlay + camera shake). Filenames contain spaces/`+` — keep helper paths in sync.
- 3D scene: over-the-shoulder framing (player back sprite near camera, enemy front sprite far), weather fog/particles, `TerrainParticles`. **Grounding**: each mon stands on a static `PlatformDisc` (frosted radial quad at y=0.01, terrain/weather-tinted) with a ground shadow that is a SIBLING of the bobbing sprite group (pinned to y=0.02, driven from the sprite's useFrame — tracks lunge/shake, shrinks at bob apex, fades on faint/capture; putting it inside the group makes it bob with the mon and look like floating). A transparent-center ground vignette disc anchors y=0 in backdrop mode; the old arena rings render in abstract mode only. Scene fog is DISABLED over backdrops (except sandstorm) — it only affects sprites and made them darker than the painted ground. `DustMotes` drift in clear/sunny weather only (never stacks with weather particles). A pointer-transparent DOM radial vignette overlays the canvas in BattleSimulator. Cinematics: `VsIntro` splash defers `beginBattle` until dismissed (AI/results effects guard on `intro === null`); Pokéball send-out on sprite entry (module-cached texture, NOT useLoader); `CameraRig` fov-only dolly on faints; move-class FX — special = `BeamEffect`, physical = `SlashEffect` + harder lunge, status = `SparkleEffect` (driven by `SceneFx.damageClass`). The arena renders a **gen-6 battle-scene backdrop** (`src/data/battleBackgrounds.ts` picks one per battle: weather/terrain-aware, random pool otherwise, Elite Four chamber for gauntlet bosses; 13 JPGs bundled in `public/assets/backgrounds/`, sourced from Pokémon Showdown). The `Backdrop` plane in `BattleScene3D` is placed along the camera's **view axis** (the r3f camera is tilted toward the origin — don't center planes on camera height) and sized cover-style preserving the 800×480 image aspect; weather multiplies a mood tint onto it. Abstract SkyDome/floor remain as the no-backdrop fallback.
- HUD: framer-motion — panel entrance slides, two-layer HP bar (spring fill + lagging white ghost), animated HP numbers, staggered post-battle panel.
- Sound: `src/utils/soundEffects.ts` — HTMLAudioElement SFX from `public/sounds/`, random battle music from `public/audio/battle/`.
- `battle/BattleLog.tsx` uses `dangerouslySetInnerHTML` for styled log messages — keep log content trusted/escaped.
