## What this app is

React 19 + TypeScript + Vite Pok√©dex/battle simulator using MUI v7, Emotion, framer-motion, and TanStack React Query v5. Installable PWA via `vite-plugin-pwa` in `vite.config.ts` (autoUpdate SW; precache = app shell only ‚Äî audio/backgrounds/particles/trainers are runtime-cached CacheFirst, PokeAPI is NetworkFirst; icons `public/pwa-*.png` are generated flat-SVG rasters, NOT the 30√ó30 `assets/items/poke-ball.png`). `registerSW` is called in `main.tsx`; `vite-env.d.ts` references `vite-plugin-pwa/client`.

## Architecture

Single component tree, no router, no global state library:

```
main.tsx  ‚Üí QueryClientProvider + MUI ThemeProvider (src/theme.ts)
App.tsx   ‚Üí <Pokemon />
Pokemon.tsx (~1500 lines) ‚Üí 4 tabs:
  Pok√©dex            (list/detail/evolutions, favorites, compare)
  TeamBuilder.tsx    (teams persisted to localStorage key "pokemonTeams")
  TypeEffectiveness.tsx
  BattleSimulator.tsx (~3000 lines; receives teams, getTypeColor, TYPE_EFFECTIVENESS as props)
```

### Battle system (the core of the app)

- **Rules engine**: `src/utils/battleEngine.ts` ‚Äî pure, immutable, rng-injectable. `resolveAction(state, action, chart, rng)` returns `{ state, events: BattleEvent[] }`; the UI never mutates battle state directly. Mechanics: stat stages (¬±6, `stageMultiplier`), speed-based round order (`beginBattle`, faster active mon leads each round; paralysis halves speed; Quick Claw can steal the lead), physical/special split (`getDamageClass`, Gen-3 type rules; special moves read special-attack/special-defense; burn weakens physical only), weather/terrain, status effects incl. confusion self-hits, energy system, abilities (Overgrow/Blaze/Torrent low-HP boost; Intimidate on-enter; Levitate immunity; Static retaliation; Sturdy survive-at-1HP), and held items (`src/data/items.ts` `HELD_ITEMS`: Leftovers round heal, type boosters, Focus Sash, Quick Claw). `BattleMon.moves` carries the moveset (defaults to type-derived). Tested in `src/utils/__tests__/` (engine, AI, movesets, progression, gauntlet, recruitment; shared fixtures in `helpers.ts`).
- **Movesets**: `src/utils/movesets.ts` ‚Äî real per-Pok√©mon level-up learnsets from PokeAPI (`getMovesetForPokemon`, cached per species + localStorage mirror `pokedexGame.movesets.v2`); any fetch failure falls back to `getMovesForTypes`. `getFullLearnset` (‚â§20 entries, mirror `pokedexGame.learnsets.v1`) feeds the TeamBuilder Move Manager (`MoveManagerDialog`, saves `MonProgress.customMoves`). `mapApiMove` also maps priority / multi-hit / flinch / stat-change debuffs. Battle start is async (`starting` flag guards double-clicks).
- **Combat depth**: priority uses a **momentum model** (a priority move makes you lead the NEXT round ‚Äî a true interrupt is impossible in the round-first turn model; precedence momentum > Quick Claw > speed in `determineRoundFirst`). Multi-hit loops per-hit damage (survive-once effects check each hit), flinch blocks a defender that hasn't acted this round (`BattleMon.flinched`, consumed on act, cleared on switch/round-end), debuffs lower opponent stages. The `'move'` event carries `damageClass` (drives scene FX). Expert AI predicts lethal threats and retreats to resistant bench mons.
- **Safari & catching**: `src/utils/safari.ts` (biomes ‚Üí type pools + backdrops, `rollWildEncounter` rarity tiers) and engine support in battleEngine.ts (`EngineState.wild/balls/caught`, `throwBall` action, pure `catchChance`). Caught mons go to the Box; ball economy on `profile.balls` (`BALLS` in data/items.ts, drops in data/rewards.ts).
- **Battle Tower**: `src/utils/tower.ts` ‚Äî level-50-normalized 3v3 streak ladder (no held items, fixed item loadout ‚Äî never persists engine items back to the profile), every 7th battle a boss with an elite ace; streaks on `records.towerStreak/towerBestStreak`.
- **Achievements**: `src/utils/achievements.ts` pure catalog + `evaluateAchievements`/`applyAchievements`, run inside useBattleResults after the profile update; earned ids on `profile.achievements`. Trainer Card tab (`TrainerCard.tsx`) renders records/dex completion/badges/achievements/save-data card/replays/Box.
- **Natures & IVs**: `src/data/natures.ts` (25 classic natures ¬±10%, `Ivs` 0‚Äì31/stat, `natureMultiplier`/`rollNature`/`rollIvs`/`natureLabel`). Persisted on `MonProgress.nature/ivs` ‚Äî rolled ONCE in `registerMonProgress` (rng-injectable last param) and backfilled once for old saves in `mergeProfile` (`usePlayerProfile.ts`, exported + tested; never rerolls filled entries). Applied in `createBattleMon`: `effectiveBase = round((base + floor(iv/2)) * natureMult * statMod)` ‚Äî HP never nature-modified; no opts ‚Üí identity. Player-side battles thread `progress.nature/ivs` (all six start handlers + hotseat P2 + tower); AI opponents stay neutral. TeamBuilder shows a nature chip + IV tooltip per slot.
- **Pok√©dex completion**: `profile.dex { seen: number[]; caught: number[] }`. Seen registers in `launchBattle` (both sides, once per battle, skipped for replays); caught registers inside `registerMonProgress` (covers catch/recruit/starter/box‚Üíteam; caught implies seen). Pure helpers in progression.ts (`registerDexSeen/registerDexCaught` ‚Äî same-ref when no change, `dexCompletion`, `KANTO_DEX_SIZE`); pre-dex saves backfill caught from mons‚à™box. UI: header chip + per-card caught/seen icons in the Pok√©dex tab, progress bars on TrainerCard (Johto rows appear once a Johto id registers), 5 dex achievements (Kanto-only thresholds).
- **Battle replays**: `src/utils/rng.ts` (`mulberry32` seeded PRNG) + `src/utils/replay.ts` (localStorage `pokedexGame.replays.v1`, newest-first cap 20, `format` stamp). The engine rng is seeded per battle (`battleRngRef` in BattleSimulator); EVERY engine call (beginBattle/resolveAction/resolveForcedSwitch ‚Äî player, AI and forced-switch paths) must use it or replays drift (replay.test.ts proves record‚Üírebuild‚Üíre-run gives identical events). Both sides' actions are recorded as `ReplayStep`s ‚Äî AI is NEVER re-run on playback (`selectAIAction` keeps Math.random). `monConfigFrom` snapshots the EFFECTIVE pokemon object (post nature/IV/statMod rewrite) so playback is exact and offline ‚Äî rebuild passes NO nature/ivs/statMod to createBattleMon (they're already folded in; passing them would double-apply). Playback: `pendingReplay` prop from Pokemon.tsx (queued by TrainerCard Watch ‚Üí Battle tab), step-driver effect paces recorded steps; `replaying` guards ALL consequences (useBattleResults, dex-seen, saving, input, AI effects). Debug weather/terrain taints the recording (not saved).
- **Hooks split**: battle presentation lives in `src/hooks/useBattleEvents.ts` (log/floating text/SceneFx/processEvents) and consequences in `src/hooks/useBattleResults.ts` (XP/drops/league/tower/catch/achievements/recruit/evolution ‚Äî applied exactly once per battle); shared UI constants in `components/battle/battleUi.ts`. BattleSimulator keeps mode glue + start handlers.
- **Cries**: `playCry(pokemonId)` in soundEffects.ts streams from the PokeAPI cries GitHub mirror (send-out, faint, Pok√©dex detail button); best-effort, silent on failure.
- **League Challenge**: `src/data/league.ts` ‚Äî 8 Kanto gyms ‚Üí Elite Four ‚Üí Champion Blue, then the **Johto post-game** (8 more gyms ‚Üí Champion Red at floor 100; gating is pure array order ‚Äî `isStageUnlocked` requires all prior stages, so Johto needs the Kanto champion win; `GYM_STAGES` is Kanto-only, `JOHTO_GYM_STAGES`/`JOHTO_STAGE_IDS` for the rest; beating Red sets `league.champion2`). Strict linear unlocks, level floors 52‚Üí100 scaling with player avg, rosters via cached `fetchLeagueTeam` (PokeAPI), XP √ó2, badges persisted in `profile.league.defeated`; Round 2 rematches (+15 levels, √ó2.5 XP, `league.defeatedRematches`) unlock at champion and stay Kanto-only. Trainer portraits bundled in `public/assets/trainers/` (Showdown). UI: `LeagueCard` in BattleSetup, portrait chip in battle, badge labels in PostBattlePanel.
- **Kanto Journey** (adventure mode): `src/data/journey.ts` ‚Äî 20-node linear path Pallet Town ‚Üí Indigo Plateau (`JOURNEY_NODES`: trainers, safari `biomeId`s, `gymIds` linking into the league ladder ‚Äî journey gym order MUST match ladder order, enforced by journey.test.ts). Progress in `profile.journey` (position/clearedTrainers/starterId); gym completion reads `league.defeated` (never duplicated). UI: `battle/JourneyMap.tsx` fullscreen dialog (starter pick incl. hidden Pikachu at level 5 ‚Üí creates the `journey-team` via `onCreateTeam` prop from Pokemon.tsx; vertical metro map). The Journey card renders via BattleSetup's `journeyCard` prop so it shows even with ZERO teams (it IS the new-player onboarding). Trainer wins append via `journeyTrainerId` in useBattleResults; travel awards Pok√© Balls.
- **Mobile (phone breakpoint `down('sm')`)**: battle scene uses portrait-aware sprite positions + fov (aspect < 1 ‚Üí x¬∑0.4/0.55, fov 56, in `SceneContents`; CatchBallEffect takes positions as props); HUD renders `MonStatusPanel dense` pills; move grid is always 2√ó2; battle log lives in a bottom-sheet Drawer (`logOpen`); everything fits 390√ó844 with zero dialog scroll. Pok√©dex detail modal stacks on xs.
- **Orchestration**: `BattleSimulator.tsx` holds React state, bridges engine events ‚Üí log/sounds/floating text/3D `SceneFx` in `processEvents`, and owns the battle modes: quick vs AI, gauntlet, and **hotseat PvP** (`opponentKind === 'human'` ‚Äî both teams take input, AI effects disabled, no XP/rewards, both sides get stock inventories). Battle pacing scales with a persisted speed setting (localStorage `battleSpeed`, 1/1.5/2√ó).
- **AI opponent**: `src/utils/battleAI.ts` ‚Äî heuristic move scoring over `mon.moves` with difficulties (beginner/intermediate/expert) and personalities (aggressive/defensive/balanced), item use and expert switching.
- **Meta-game** (all pure + tested): `src/utils/progression.ts` (XP curve from level 50, `PlayerProfile` with mons/items/heldItems/records/box/dex persisted via `src/hooks/usePlayerProfile.ts` to localStorage `pokedexGame.profile.v1`; schema backfill lives in the exported pure `mergeProfile`; the hook is instantiated ONCE in `Pokemon.tsx` and passed as props to TeamBuilder + BattleSimulator ‚Äî never instantiate it twice), `src/data/rewards.ts` (post-win item drops + rare held-item drops), `src/utils/gauntlet.ts` (endless escalating stages, HP carry-over, boss every 3rd stage with elite shiny mons), `src/utils/recruitment.ts` (post-win recruit offers, elite/legendary rolls), `src/utils/evolution.ts` (PokeAPI chain-walk, level-up evolutions; evolving transfers nature/ivs/heldItem). TeamBuilder hosts the **Box manager** (Available | Box tabs; add-to-team/release/drag) plus per-slot level/shiny/elite/nature badges and held-item equipping.
- **Save safety**: `src/utils/saveData.ts` ‚Äî `SAVE_KEYS` (every localStorage key the game writes; ADD NEW KEYS HERE), `buildFullSave`/`parseFullSave`/`applyFullSave` (injectable storage, tested). TrainerCard's Save Data card does export (Blob download) and import (file input ‚Üí typed validation ‚Üí confirm ‚Üí apply ‚Üí `window.location.reload()` so the single profile hook rehydrates).
- **Battle UI components** in `src/components/battle/`: `BattleScene3D` (r3f arena), `BattleSetup` (team select + records + gauntlet card), `PostBattlePanel` (results, XP bars, drops, recruit/evolution offers), `MoveSelection`, `BattleLog`, `TerrainParticles`, `RecruitOfferCard`, `EvolutionPrompt`.
- Teams changes flow through `Pokemon.tsx` handlers (`onAddPokemonToTeam`, `onEvolvePokemon`) ‚Äî it owns `teams` and the `pokemonTeams` localStorage persistence.

### Data sources

- Pok√©mon data comes live from PokeAPI (`https://pokeapi.co/api/v2/`) via `fetch` + React Query inside `Pokemon.tsx` (infinite query for the list; separate queries/fetches for species, evolution chains, forms). No local Pok√©mon dataset.
- Battle sprites come from the PokeAPI sprites CDN via `src/utils/spriteSources.ts` ‚Äî URLs pinned to a commit SHA (never `master`; bump `SPRITES_SHA` deliberately). Fallback ladder: showdown GIF ‚Üí gen-5 animated ‚Üí CDN static ‚Üí **bundled local static** (`public/assets/sprites/{id}.png` + `back/{id}.png`, gen 1‚Äì2 only via `localStaticSprite` ‚Äî same-origin, cannot fail; excluded from SW precache, runtime-cached) ‚Üí `front_default`. `src/hooks/useAnimatedTexture.ts` decodes GIFs with gifuct-js into three.js CanvasTextures and **retries the whole ladder with backoff (1s/3s/8s)** when every rung fails ‚Äî without this a transient network blip left the sprite invisible for the entire battle. `<img>` fallbacks (BattleSetup preview, TeamRoster) MUST be one-shot with a `dataset` guard: React's synthetic `onError` refires even after `img.onerror = null`, and re-setting `src` to the same failing URL restarts the load ‚Üí infinite request loop.
- Moves are single-sourced in `src/data/moves.ts`, keyed by type; a Pok√©mon's moveset is derived from its types (fallback: Tackle). Items live in `src/data/items.ts`, `TYPE_EFFECTIVENESS` in `src/data/typeChart.ts` (a copy still exists in `Pokemon.tsx`/`TeamBuilder.tsx` ‚Äî passed as props).

### Presentation subsystems

- Move impact FX: `src/utils/moveAnimationHelper.ts` maps all 18 move types ‚Üí grid spritesheets under `public/assets/particles/Spritesheets/` (some reused with tints); rendered by `ImpactEffect` inside `battle/BattleScene3D.tsx` (texture-offset animation, additive blending; critical hits add a gold burst overlay + camera shake). Filenames contain spaces/`+` ‚Äî keep helper paths in sync.
- 3D scene: over-the-shoulder framing (player back sprite near camera, enemy front sprite far), weather fog/particles, `TerrainParticles`. Cinematics: `VsIntro` splash defers `beginBattle` until dismissed (AI/results effects guard on `intro === null`); Pok√©ball send-out on sprite entry (module-cached texture, NOT useLoader); `CameraRig` fov-only dolly on faints; move-class FX ‚Äî special = `BeamEffect`, physical = `SlashEffect` + harder lunge, status = `SparkleEffect` (driven by `SceneFx.damageClass`). The arena renders a **gen-6 battle-scene backdrop** (`src/data/battleBackgrounds.ts` picks one per battle: weather/terrain-aware, random pool otherwise, Elite Four chamber for gauntlet bosses; 13 JPGs bundled in `public/assets/backgrounds/`, sourced from Pok√©mon Showdown). The `Backdrop` plane in `BattleScene3D` is placed along the camera's **view axis** (the r3f camera is tilted toward the origin ‚Äî don't center planes on camera height) and sized cover-style preserving the 800√ó480 image aspect; weather multiplies a mood tint onto it. Abstract SkyDome/floor remain as the no-backdrop fallback.
- HUD: framer-motion ‚Äî panel entrance slides, two-layer HP bar (spring fill + lagging white ghost), animated HP numbers, staggered post-battle panel.
- Sound: `src/utils/soundEffects.ts` ‚Äî HTMLAudioElement SFX from `public/sounds/`, random battle music from `public/audio/battle/`.
- `battle/BattleLog.tsx` uses `dangerouslySetInnerHTML` for styled log messages ‚Äî keep log content trusted/escaped.
